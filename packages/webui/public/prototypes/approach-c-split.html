CI-Gated Pipeline: PR gate enforcement before merge
HIGH agent · 7h ago
Description
Block merge to main unless CI pipeline passes all required checks including type-check, lint, and integration tests. Currently, PRs can be merged without any automated verification, which has led to several broken builds landing on main.

This should integrate with the existing GitHub Actions workflow and enforce branch protection rules. Required checks should include: TypeScript compilation, ESLint, unit tests, and the integration test suite. Failed checks should block the merge button and post a clear status comment.

Consider also adding a "bypass" mechanism for emergency hotfixes that requires explicit approval from a designated reviewer, so we don't get stuck when CI is flaky.
