# Skill Sync: Launchd-based Team Skill Distribution

## Problem

Claude Code skills live in `~/.claude/skills/` per-user. When skills are updated or new ones are added, each team member must manually copy them. Chris doesn't know when skills change and shouldn't have to.

## Solution

A launchd agent that runs every 15 minutes, pulls the latest zazigv2 repo, and symlinks shared skills from the repo's `.claude/skills/` into each user's `~/.claude/skills/`.

## Design

### Architecture

```
zazigv2 repo (.claude/skills/)     ← source of truth
        |
        | git pull --ff-only (every 15 min)
        |
        v
~/.claude/skills/cardify -> ~/Documents/GitHub/zazigv2/.claude/skills/cardify
~/.claude/skills/ship    -> ~/Documents/GitHub/zazigv2/.claude/skills/ship
~/.claude/skills/my-personal-skill/  (untouched, real directory)
```

### Key decisions

- **One-way sync**: repo → local. Skills are edited in the repo, never locally.
- **Symlinks, not copies**: `ln -sfn` from `~/.claude/skills/{name}` to `{repo}/.claude/skills/{name}`. Git pull updates the source, symlinks point to latest automatically.
- **Personal skill protection**: if a real directory (not a symlink) exists at `~/.claude/skills/{name}`, the sync skips it. Personal skills are never overwritten.
- **Repo path auto-detected**: the install script derives the repo path from its own location. No hardcoding, no config file.

### Files

| File | Purpose |
|------|---------|
| `scripts/skill-sync.sh` | The sync script launchd runs every 15 min |
| `scripts/install-skill-sync.sh` | One-time setup: writes plist, loads launchd agent, runs first sync |
| `~/Library/LaunchAgents/com.zazig.skill-sync.plist` | Generated by install script |

### skill-sync.sh

```bash
#!/bin/bash
set -euo pipefail

REPO_DIR="${ZAZIG_REPO_DIR:?ZAZIG_REPO_DIR not set}"
SKILLS_SRC="$REPO_DIR/.claude/skills"
SKILLS_DST="$HOME/.claude/skills"
LOG="/tmp/zazig-skill-sync.log"

log() { echo "$(date +%Y-%m-%dT%H:%M:%S) $1" >> "$LOG"; }

# Verify repo exists
if [ ! -d "$REPO_DIR/.git" ]; then
  log "ERROR: repo not found at $REPO_DIR"
  exit 1
fi

# Pull latest (silent, fail-safe)
cd "$REPO_DIR"
if ! git pull --ff-only origin master >> "$LOG" 2>&1; then
  log "SKIP: git pull failed (dirty tree or no network)"
  exit 0
fi

# Ensure target directory exists
mkdir -p "$SKILLS_DST"

# Symlink each skill
for skill_dir in "$SKILLS_SRC"/*/; do
  [ -d "$skill_dir" ] || continue
  name=$(basename "$skill_dir")
  target="$SKILLS_DST/$name"

  # Skip personal skills (real directories, not symlinks)
  if [ -d "$target" ] && [ ! -L "$target" ]; then
    log "SKIP: $name (personal skill, not a symlink)"
    continue
  fi

  # Create or update symlink
  ln -sfn "$(cd "$skill_dir" && pwd)" "$target"
done

log "OK: synced $(ls -d "$SKILLS_SRC"/*/ 2>/dev/null | wc -l | tr -d ' ') skills"
```

### install-skill-sync.sh

```bash
#!/bin/bash
set -euo pipefail

# Detect repo root from this script's location
SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
REPO_DIR="$(cd "$SCRIPT_DIR/.." && pwd)"

# Verify this is the right repo
if [ ! -f "$REPO_DIR/CLAUDE.md" ]; then
  echo "ERROR: Can't find zazigv2 repo root. Run from scripts/ directory."
  exit 1
fi

PLIST_NAME="com.zazig.skill-sync"
PLIST_PATH="$HOME/Library/LaunchAgents/$PLIST_NAME.plist"

echo "Installing skill-sync launchd agent..."
echo "  Repo: $REPO_DIR"
echo "  Plist: $PLIST_PATH"

# Write plist
cat > "$PLIST_PATH" <<EOF
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>$PLIST_NAME</string>
    <key>ProgramArguments</key>
    <array>
        <string>/bin/bash</string>
        <string>$REPO_DIR/scripts/skill-sync.sh</string>
    </array>
    <key>StartInterval</key>
    <integer>900</integer>
    <key>RunAtLoad</key>
    <true/>
    <key>StandardOutPath</key>
    <string>/tmp/zazig-skill-sync.log</string>
    <key>StandardErrorPath</key>
    <string>/tmp/zazig-skill-sync.log</string>
    <key>EnvironmentVariables</key>
    <dict>
        <key>PATH</key>
        <string>/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin</string>
        <key>HOME</key>
        <string>$HOME</string>
        <key>ZAZIG_REPO_DIR</key>
        <string>$REPO_DIR</string>
    </dict>
</dict>
</plist>
EOF

# Load the agent
launchctl unload "$PLIST_PATH" 2>/dev/null || true
launchctl load "$PLIST_PATH"

# Run first sync immediately
echo "Running first sync..."
ZAZIG_REPO_DIR="$REPO_DIR" bash "$REPO_DIR/scripts/skill-sync.sh"

echo "Done. Skills will sync every 15 minutes."
echo "Log: /tmp/zazig-skill-sync.log"
```

### Error handling

| Scenario | Behavior |
|----------|----------|
| No network | `git pull --ff-only` fails, skip cycle, try in 15 min |
| Dirty working tree | `--ff-only` refuses to merge, skip cycle |
| Repo moved/deleted | Log error, exit 1 |
| Personal skill conflict | Skip that skill, log warning |
| New skill in repo | Symlink created automatically |
| Skill removed from repo | Symlink becomes dangling — harmless, Claude Code ignores it |

### Onboarding

Chris (or any new team member) runs one command:

```bash
cd ~/path/to/zazigv2
./scripts/install-skill-sync.sh
```

After that, skills auto-sync. New skills appear within 15 minutes of being pushed to master.

### Uninstall

```bash
launchctl unload ~/Library/LaunchAgents/com.zazig.skill-sync.plist
rm ~/Library/LaunchAgents/com.zazig.skill-sync.plist
# Optionally remove symlinks:
# find ~/.claude/skills -type l -delete
```
