The Six Layers

Every zazig worker is assembled from up to six layers, compiled by the orchestrator into a prompt stack at dispatch time. The agent never sees the raw config — it receives a compiled prompt fragment.

Layer 1
Personality
Who they are. Bounded numeric dimensions + archetype philosophy. Compiled into natural-language prompt.
Layer 2
Role Prompt
What they do. Operational scope, responsibilities, hard stops, output contract.
Layer 3
Skills
How they work. Procedural capability files loaded per job from ~/.claude/skills/.
Layer 4
Doctrines
What they believe. Role-specific heuristics, frameworks, policies. Can contradict across roles.
Layer 5
Canons
What they've studied. Shared reference knowledge — books, regulations, playbooks. Company-wide.
Layer 6
Memory
What they remember. Episodic context from past jobs, learned patterns, project-specific knowledge.

Three Tiers of Workers

Modeled on how real companies work. Each tier gets a different subset of the six layers and different operational privileges.

Executives

  • Personality: Full — 9 dims, archetype, philosophy
  • Knowledge: Wide domain doctrines + all canons
  • Memory: Persistent across all jobs
  • Heartbeat: Yes — autonomous
  • Gateway: Bidirectional (Slack, etc.)
  • Charter: Full mandates + interdictions
  • Autonomy: High
  • Managed by: Founders

CPO, CTO, CMO, CEO, VP-Eng

Employees

  • Personality: Lite — values + 2-3 dims
  • Knowledge: Deep, narrow doctrines + relevant canons
  • Memory: Persistent — builds expertise
  • Heartbeat: Yes — routine cycles
  • Gateway: Scoped, write-only
  • Charter: Light — interdictions only
  • Autonomy: Moderate
  • Managed by: Executives

Sr. Engineer, PR Reviewer, Researcher

Contractors

  • Personality: None — values-only
  • Knowledge: Deep specialist doctrines + specialist canons
  • Memory: Job-scoped + optional shared
  • Heartbeat: No
  • Gateway: None
  • Charter: None — task-scoped
  • Autonomy: None
  • Managed by: Dispatched per job

Security Tester, Legal Reviewer, Migration Specialist

The Prompt Stack

Assembly order is informed by "Lost in the Middle" research — LLMs attend most to the beginning and end of context. Static prefix is cached across jobs (~10x cost reduction).

1
Personality prompt — who you are (highest attention)
2
Role prompt — what you do
3
Doctrine Tier 1 index — what you believe (stable pointers)
3b
Canon library pointers — what you've studied (stable pointers)
cache break line
4
Proactive doctrine claims — task-relevant beliefs
5
Doctrine tension blocks — cross-role contradictions
6
Canon source summaries — if high-similarity match
7
Skill content — how to work on this type of task
8
Task context — what to do now
9
Memory context — what you remember

Positions 1–3b form a static prefix cached across all jobs for the same role within a company. Positions 4–9 are dynamic and change per task.
