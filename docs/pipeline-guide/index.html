Step 1

Capture: Getting Ideas In

Record ideas through any channel. Your CPO captures them instantly. Voice notes, text messages, terminal conversations — all end up in the same inbox.

💻
Terminal
Talk to your CPO directly. It captures ideas mid-conversation without breaking flow.
✈️
Telegram
Send text or voice notes on the go. Voice gets transcribed automatically. Multi-idea messages get split.
💬
Slack
Messages route into the ideas inbox. Your team's existing communication channel feeds the pipeline.
🤖
Agent-discovered
Monitoring agents spot opportunities, research viability, and submit structured ideas automatically.

What gets recorded

Every idea captures: raw_text (your exact words, never modified), originator (who said it), source (which channel), and a title + description added by processing.

Steps 2 & 3

Processing & Triage

Raw input gets cleaned automatically. Then your CPO reviews every idea and recommends what to do with it. You make the final call.

⚙️

Ideaify processes automatically

  • Splits multi-idea inputs into individual records
  • Cleans each into a clear title + description
  • Categorises by scope, complexity, and domain
  • Flags anything ambiguous for clarification

Ideaify does NOT decide if an idea is good or bad. That's the CPO's job.

🔍

CPO triages every sweep

  • Reviews title, description, flags, and notes
  • Refines description and adds tags
  • Sets priority: low / medium / high / urgent
  • Recommends: promote, park, or reject

Your CPO never promotes without your explicit approval.

Idea Lifecycle

new → triaged → promoted → becomes a feature, job, or research task
→ parked not now, maybe later — reviewed periodically
→ rejected not doing this — preserved for context

Promotion paths

→ Feature

Worth building. Creates a feature record, enters the spec pipeline. Most ideas go here.

→ Job

Small standalone task. Goes straight to the build queue. Quick fixes, one-off changes.

→ Research

Interesting but needs investigation. A research contractor digs deeper, may produce a proposal.

Steps 4–7

The Build Pipeline

Once an idea is promoted to a feature, it moves through four automated stages. You stay involved for spec review and final verification.

04
Spec

CPO writes the feature spec

Using the /spec-feature skill, your CPO drafts:

  • Description — what the feature does, in plain language
  • Acceptance criteria — how you'll know it works
  • Human checklist — what you verify manually on the test server

You review and iterate until the spec is right. Then it's marked ready for breakdown.

05
Breakdown

Feature gets decomposed into jobs

The Breakdown Specialist (an ephemeral contractor) runs jobify to produce 3–8 executable jobs with dependency graphs.

  • Each job gets a self-contained spec
  • Gherkin acceptance tests (machine-verifiable)
  • Complexity rating and dependency ordering

Fully automated. You don't need to do anything here.

06
Build

Workers execute jobs

The orchestrator dispatches jobs to available worker slots. Each job runs in an isolated git worktree:

  • Clone repo into isolated workspace
  • Read spec and acceptance tests
  • Write code, run tests, push to feature branch
  • Jobs execute in dependency order, parallel where possible

When all jobs complete, results combine into a single branch.

07
Ship

Verify and merge

Two verification gates before shipping:

  • Code review — multi-agent review (security, performance, architecture, simplicity)
  • Feature verification — your human checklist tested on staging

When verification passes, a PR is created. You or your CPO merge it.
