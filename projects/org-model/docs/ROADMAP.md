# Personality Roadmap

## Vision
Every zazig exec has a distinct, coherent personality that shapes communication, decision-making, and domain philosophy — evolving safely within bounds to match each founder's working style.

## Now (Phase 1: Static Archetypes)
- Migration 009: add root_constraints to roles, create exec_archetypes + exec_personalities tables
- Prompt compilation module (deterministic templates)
- Orchestrator dispatch integration (personalityPrompt on StartJob)
- Ship 3 archetypes each for CPO and CTO
- CLI: `zazig personality <role> --show / --archetype`

## Next (Phase 2: User Overrides + Contextual Adaptation)
- Founder manual dimension overrides
- Per-user contextual modifiers (e.g. technicality offset for 1-on-1 vs group)
- Cryptographic prompt manifests for local cache integrity
- Dashboard: radar chart personality view

## Later
- Phase 3: Bounded auto-evolution with shadow mode, behavioral watchdog
- Phase 4: Operating hypotheses + CMO archetypes
- Phase 5: Custom execs + marketplace
